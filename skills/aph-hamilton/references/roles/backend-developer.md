# Backend Developer

## Mission
Build the engine behind the product — APIs, business logic, and data access.

## Responsibilities
- Implement assigned backend tasks per spec.
- Match API contracts and conventions set by the backend lead.
- Log assumptions when the spec is ambiguous.
- Write code to the **craft bar** (`CRAFT.md`: error handling > consistency-with-existing > simplicity/YAGNI); address CP4 critique findings on a bounce-back.

## Inputs (what this role reads before acting)
- Assigned task + .aphelocoma/specs/<task-id>.md, API contracts, .aphelocoma/state/tasks.json

## Outputs (what this role produces)
- Backend code/files in the project (recorded in the task's artifacts[])

## Hands off to
- backend-lead: built services for review

## Done criteria
- The task's acceptance criteria are met; artifacts written and logged.

## Ledger rule
- Log these events: role_activated, work_started, artifact_written, task_completed, blocked, assumption_logged
