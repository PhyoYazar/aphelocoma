# T1 — Todo list app (single file)

## Goal
Create a simple todo list app as ONE self-contained file under `product/` that lets a
user add, view, complete, and remove todo items.

## Scope / Non-scope
**In scope**
- A single file under `product/` implementing the todo list.
- The four core operations, working in the browser: add, list/view, complete, remove.

**Out of scope**
- Multiple files, build tooling, or external dependencies.
- Persistence beyond the page session, authentication, multi-user support, styling polish.
- Any non-trivial application code beyond a small, working single-file implementation.

## Interfaces / files touched
- `product/todo.html` — the single self-contained file (the entire product).

## Acceptance criteria
- [ ] Exactly one product file exists under `product/` (`product/todo.html`).
- [ ] The file is self-contained: no build step and no external dependencies required.
- [ ] The file identifies the app as a todo list and implements all four core
      operations: add, list/view, complete, remove.
- [ ] The file's path is recorded in task T1's `artifacts[]` in `state/tasks.json`.
